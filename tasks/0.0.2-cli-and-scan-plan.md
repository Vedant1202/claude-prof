## Overview

Strengthen the CLI foundation, then ship the first new command through it. One enabler (a zero-dep
**command registry**) plus two features: CLI-standards polish (per-command help, uniform `--json`,
stdout/stderr split + `--quiet`, completions) and `cprof scan` (standalone secret gate reusing
`leak-check.ts`). Rollback / ledger v2 / drift are out of scope (next spec).

## Architecture decisions (from spec §4)

- **D1 — command registry, zero-dep.** A declarative `Command[]` table drives dispatch, help, and
  completions from one source. Keep each command's `run(flags, opts)` signature and its own
  positional parsing; no CLI framework. The Task 1 refactor is **behavior-preserving** — the existing
  CLI suite must stay green unchanged.
- **D3 — `--json` envelope.** Every command emits one stdout object `{ command, ok, … }`. No
  schema-stability promise in alpha.
- **D4 — defer color.** Land stdout/stderr split + `--quiet` only; no color machinery yet (nothing to
  gate today).
- **D5 — `scan` takes files**, raw-byte scan, no dir/glob walking.
- **D6 — exit codes** 0 clean / 3 leak / 2 not-found / 1 usage; findings carry `line:col`.
- **D7 — ship `scan` as-is**, documented "same limits as redaction," `--strict` held in reserve.

## Grounded current state (recon)

- Dispatch is an `if`-chain in `main()` ([cli/src/index.ts:85](../packages/cli/src/index.ts)); command
  list lives only in the `USAGE` string (lines 22–43). `--help`/`-h` is matched **anywhere** before
  dispatch (lines 73–81) — the per-command-help bug.
- Tests call `main(argv, { cwd, homeDir, stdout, stderr, env })` with `createWritable()`; key
  invariants to preserve: `--help` overview contains `"Usage: cprof <command>"` + `"init"` + `"install"`;
  unknown command → exit 1 + `"unknown command: <x>"` + `"--help"` ([cli/test/index.test.ts:36–61](../packages/cli/test/index.test.ts)).
- `--json` today: `validate`/`diff`/`profiles` only (ad-hoc `flags.includes("--json")`).
- Status logs on stdout to relocate: init `"Wrote claude-profile.json"`
  ([commands/init.ts:73](../packages/cli/src/commands/init.ts)), refresh `"Refreshed…"`, install
  `"Installed N writes"`.
- `leak-check.ts` `OutputLeak` = `{path, tokenIndex, reason}`; token-scan matches expose `match.index`
  (byte offset → line:col); the provider-pattern hit (`tokenIndex: -1`) has **no** offset → position
  is best-effort for that class.

### Phase 2 — Help & stream hygiene

#### Task 2: Per-subcommand help (D2)
**Description:** Stop `main()` from swallowing `--help` anywhere. Treat `--help`/`-h` as global only

**Acceptance criteria:**
- [ ] `cprof init --help` prints init's usage (not the overview), exit 0; same for each command.
- [ ] `cprof help diff` prints diff's usage; `cprof --help` overview unchanged.
- [ ] Per-command `usage` strings live in the registry entry (single source).

**Verification:**
- [ ] New tests assert each command's `--help` shows its own usage; overview/unknown-command tests still green.
- [ ] `corepack pnpm --filter @cprof/cli test`.

**Dependencies:** T1 · **Files:** `cli/src/index.ts`, `cli/src/registry.ts`, `cli/test/help.test.ts` (new) · **Scope:** S

#### Task 3: Stream hygiene + `--quiet` + `parseCommonFlags` (D4)
**Description:** Add `parseCommonFlags(flags) → {json, quiet, rest}` (each command calls it first, then

**Acceptance criteria:**
- [ ] `init`/`refresh`/`install` success status goes to stderr; stdout carries no status text on success.
- [ ] `--quiet` silences status logs but not errors; all exit codes identical to today.
- [ ] `parseCommonFlags` strips `--json`/`--quiet` before each command's own flag parsing.

**Verification:**
- [ ] Tests assert status text appears on stderr (not stdout), and `--quiet` removes it; error paths unchanged.
- [ ] `corepack pnpm --filter @cprof/cli test`.

**Dependencies:** T1 · **Files:** `cli/src/command-utils.ts`, `cli/src/commands/{init,refresh,install}.ts`, `cli/test/streams.test.ts` (new) · **Scope:** M

## Risks and mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Registry refactor breaks a command | High | T1 is behavior-preserving; existing suite is the guard (Checkpoint A); land first (fail fast). |
| stdout/stderr move breaks consumers parsing stdout | Med | Alpha; `--json` is the stable contract; note the init "Wrote…"→stderr change in commit/changelog. |
| `scan` false positives on real configs | Med | Ship documented "same limits as redaction"; corpus sanity-check; `--strict` escape hatch held in reserve (not built now). |
| line:col off-by-one | Low | Unit tests on known offsets; keep `{ok,leaks}` shape additive so callers are safe. |
| Windows CI CRLF failure | Med | Merge `docs-followups` (`.gitattributes`) before/with this PR. |
